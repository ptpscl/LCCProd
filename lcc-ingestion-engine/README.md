# LCC Ingestion Engine

A standalone Python backend service. Its job is to read raw CSVs uploaded to Supabase Storage and ingest their rows into a Supabase Postgres Bronze table.

## Local Setup

1. Create a virtual environment:
   ```bash
   python3 -m venv venv
   source venv/bin/activate
   ```

2. Install dependencies:
   ```bash
   pip install -r requirements.txt
   ```

3. Configure environment variables:
   Copy `.env.example` to `.env` and fill in the values:
   ```bash
   cp .env.example .env
   ```

4. Run the development server:
   ```bash
   uvicorn app.main:app --reload
   ```

5. Test the health endpoint:
   Open http://localhost:8000/health in your browser, it should return `{"status": "ok"}`.

## Supabase Schema Requirements

Before running ingestion, create the target Bronze table in your Supabase database using the SQL Editor:

```sql
CREATE TABLE bronze_loyalty_sales (
    id bigint generated by default as identity primary key,
    "DATE" text,
    "TRANSACTION_NUMBER" text,
    "REGISTER_NUMBER" text,
    "STORE_CODE" text,
    "STORE_CATEGORIZATION" text,
    "CUSTOMER_NUMBER" text,
    "SKU_CODE" text,
    "TRANSACTION_TYPE" text,
    "LOYALTY_SALES" text,
    "QTY_SOLD" text,
    source_batch_id text,
    loaded_at timestamptz default now()
);

CREATE INDEX idx_bronze_loyalty_sales_date ON bronze_loyalty_sales ("DATE");
CREATE INDEX idx_bronze_loyalty_sales_store_code ON bronze_loyalty_sales ("STORE_CODE");
CREATE INDEX idx_bronze_loyalty_sales_sku_code ON bronze_loyalty_sales ("SKU_CODE");
CREATE INDEX idx_bronze_loyalty_sales_source_batch_id ON bronze_loyalty_sales (source_batch_id);

ALTER TABLE bronze_loyalty_sales ENABLE ROW LEVEL SECURITY;
CREATE POLICY "Allow all operations during pilot" ON bronze_loyalty_sales FOR ALL USING (true) WITH CHECK (true);
```

## Railway Deployment

1. Set the runtime via Nixpacks or Railway UI to Python 3.11.
2. Required environment variables in Railway:
   - `SUPABASE_URL`: Your Supabase project URL.
   - `SUPABASE_SERVICE_KEY`: Your Supabase **service_role** key (do not use the anon key, as this service needs to write to `loyalty_batches` and `bronze_loyalty_sales`).
3. This service is completely stateless — no persistent volume is required. Supabase Storage and Postgres serve as the source of truth, not local disk.
